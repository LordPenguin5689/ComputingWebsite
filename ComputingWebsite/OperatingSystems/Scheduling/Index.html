We need to know the pros and cons of the main scheduling algorithms. These are Round Robin, First Come First Serve, Shortest Job First, Shortest Time Remaining and Multi-level Feedback Queue.

"Round Robin":

In round robin scheduling, processor time is divided equally among all running tasks. Each time period is called a time slice. Each process gets a time slice, and if the process has not finished when this time is up, it stops running and the computer switches to the next process. The process that has been suspended will only be able to resume running when it is next allocated processor time.

    Pros:

  • Ensures that every task is allocated processor time without a long wait.

    Cons:

  • It does not scale well, because as more processes run, the time slice for each process gets smaller, which means that more urgent tasks are less likely to be completed quickly.

"First Come, First Server":

In first come, first served scheduling, processes run from start to finish in the order in which the processor receives their request to run. You could compare the first come, first served scheduling algorithm to a queue at a supermarket till â€” customers are served in the order in which they arrive at the till.

    Pros:

  • This is the simplest method to implement
  • Once a job starts, it will definitely complete(with other scheduling algorithms, starvation may occur ).

    Cons:

  • The processes run in the order in which the processor receives their request to run, there can be a long wait before a process can run.

In a system that only has a few concurrent processes, first come, first served can work well.

"Shortest Job First":

In shortest job first scheduling, processes are queued and the process that needs the shortest total time to complete goes first. You could compare this a supermarket queue in which the person with the fewest items in their basket is allowed to pay for their goods first.

    Pros:

  • This reduces wait times because shorter processes are removed quickly, which means that the processor can allocate more time to longer jobs.

    Cons:

  • This type of scheduling is susceptible to starvation if shorter jobs keep being added
  • The scheduler will keep prioritising these jobs, which means that longer processes will not be serviced.

"Shortest Time Remaining":

In shortest time remaining scheduling, the processes with the shortest time to completion go first, until a different task is added with a shorter time to completion, which then takes priority.

This is different to the shortest job first algorithm because a long job which is mostly complete might have a very short time remaining, and would therefore be prioritised.

    Pros:

  • Minimizes the total amount of "lateness" of tasks.
  • In situations where you cannot be late, this optimises for that.

    Cons:

  • Like shortest job first scheduling, this algorithm also has the effect of delaying longer processes from completing sooner if shorter processes are added in the meantime

"Multi-level Feedback Queue":

The multi-level feedback queue algorithm uses queues of tasks, where each queue has a different level of priority. Processes are chosen from different queues based on the priority of each queue and how much processing time each process has already had.

    Pros:

  • All tasks are given processor time, but the more important jobs are dealt with sooner.
  • The queues become more balanced the more the tasks are processed, and this ensures that no single high-priority task dominates the processor's time.

    Cons:

  • Very hard to implement.
  • It's not flexible.
  • It also requires some means of selecting values for all the parameters to define the best scheduler, thus it is also the most complex.
