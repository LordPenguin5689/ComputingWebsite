A von Neumann computer architecutre uses the fetch-execute cycle with a single control unit.

A co-processor (e.g a GPU) can be useful for complex calculations for scientific research as it is more efficient than a CPU for parallel processing of large amounts of data.

In the case of printing personalised cards for customers as part of a advertising campaign, a co-processor is not suitable for the task as there are no calculations.

CISC (Complex Instruction Set Computing)

  • Multiple machine cycles per instruction
  • Has more complex hardware
  • Physically larger in size and requires more silicon to make (and thus is more expensive)
  • Uses more energy
  • Better for more intensive tasks that require a lot of processing
  • Many ways to address memory
  • Used in laptops and Desktop PCs, made by intel or AMD

RISC (Reduced Instruction Set Computing)

  • Single machine cycle per instruction
  • Has simpler hardware
  • Physically smaller in size as less circuitry required, less silicon needed (and thus is cheaper)
  • Uses less energy
  • Runs at lower clock speed, but faster for simple tasks
  • Fewer ways to address memory
  • Used in smartphones and tablets, based around ARM processors.
